# Mechanize Take-Home Interview

Thanks for interviewing with Mechanize! This repo contains the Mechanize take-home interview assignment.

1. Spend **up to 3 hours** working on this assignment.
2. Once complete, notify us to schedule a **1 hour** video call to discuss your solution.

# Evaluation

Here is how we will evaluate the project:

- **Design and Problem Solving:** Were relevant tradeoffs considered and can design decisions be justified in terms of those tradeoffs?
- **Code Quality:** Is the code written in a way that's easy to extend, maintain, and understand?
- **Testing:** Does the level of testing give us confidence that the code works as intended? Are the right things tested, and are they tested at the right level of abstraction?
- **Styling:** Pixel‑perfect styling is not required. Focus on functionality and a sensible layout first. If there's time to make it beautiful, that's a bonus.
- **Progress:** Other things equal, more progress is better.
  These criteria are vague on purpose. We want to understand what you think is important and how you prioritize.

# Guidelines

- **Tech Stack:** This repo contains basic scaffolding for a Python backend and React frontend. This is for your convenience. If you prefer a different set of tools or stack, feel free to use that instead. We want to see what you can produce using the tools you're most comfortable with. That said, please choose something that demonstrates your software development skills. For example, implementing the task with Jekyll will make it very hard for us to understand your skillset.
- **AI:** Feel free to use whatever AI tooling you want. If you can one-shot this entire project, more power to you. Just remember that, during the video call, you will be asked to demonstrate an understanding of the code.

# Task

Without further ado, here is the task:
Create a blogging platform with the following minimum requirements:

- CRUD for posts (create, read, update, delete).
- A markdown editor for creating and editing posts.
- Posts are accessible via unique URLs (proper frontend routing).
- Functional, non-broken styling.

Once you complete this, please use any remaining time to build out additional functionality which makes the platform more usable. This interview is intentionally open-ended. Here are some ideas, in no particular order:

- Authentication and multi-author support
- Drafts / publish workflow
- Commenting system
- Pagination
- Image uploads
- Author profiles (view posts by author)
- Comment threading
- Upvotes/downvotes on posts or comments
- Favorites/bookmarks
- Search
